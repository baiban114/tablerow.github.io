博客搭建

​ 哦我的上帝啊，瞧瞧这hugo！ko no sei gai还有更难用的静态网站生成器吗？当然我也没用过其他的。

​ 相比之下hexo是中国台湾人创建的，有中文文档，就很爽。

​ 如果重命名content下的md文件，hugo编译时不会把原来的文件删掉。

​ 一开始尝试在centos7上使用hugo（linux64位0.84.x版本），并用nginx做服务器部署生成的静态页面，遇到了路径问题，样式文件加载不出来；然后修改到一半，新的静态页面突然就生不出来了。。。QAQ

​ 改了配置文件、添加了新文章、执行hugo命令后也没有生成新的、正确的静态页面。根据提示，安装extended版本 ，结果又报错说缺少c++的新版动态库，但我的确是新版了（跟CSDN上的博客比）。

​ 最后用了windows版+github pages。省时又省心，效果还一样。之前尝试部署到服务器上的我简直是个可爱的小傻瓜。

​ 另，gitee的Pages服务这几个月关闭了，预计8月左右恢复。说是为了建设绿色网络环境什么的，可能不符合规定的要被清除？

​ 近日正好收到一篇公众号的推送，关于hugo搭建博客的，就用上了。最早了解的是wordPress，但没有使用；身边人用的最多的是hexo。年轻时总想着搞个人博客，早就接触到了建站容易的静态博客，但感觉搭建起来对技术提高帮助不大，就没搞。实际上现在搞了也没多大的帮助，只是偶尔刺激自己在博客上整理思想、经验。

​ 尝试了下集成utteranc评论功能，虽然所用的diary主题没有支持，但我自己把它添上去了。

​ 现在博客内容主要是写给自己的，到不用担心信息孤岛的问题。

为什么搞静态博客？跟风

​ 雀食跟风，我明明没啥要写的，也没时间写；只是因为好多人都在搞博客。要说我也不喜欢静态博客，它搭建成本低，但定制需要额外学习成本（学习到的技术不是很通用）。它适合那些专门写文章的人，专注于写作，不用花过多心思管理代码和服务器。嘛，不过有人喜欢用就是了，没必要当成普世的目标。身边搞博客的人越来越多，还大肆吹嘘博客的好处，仿佛每个成功男人的背后都有一个静态博客网站。三人成虎，最后我也跟风搞了。前前后后花了珍贵的两天复习时间，做了一件平平无奇的事情。

​ 我倒是不反对多体验多经历，可是哪儿有这个时间呢？每一次挑战都是有成本的，每一次看似成功的选择都有可能孕育出失败的种子。开元盛世之后就是安史之变，战无不胜的拿破仑遭遇了滑铁卢，摩托罗拉在销量最高的一年见证了智能手机的诞生，而苏联在如日中天时进军了阿富汗。我慢慢变得不是很喜欢夸人，因为我看到了不少小小的辉煌和小小的失败。我不否认别人的出彩之处，只是暂时厌倦了举轻若重，有些人看见别人有前途就赶紧夸一下，生怕落后。按说出于礼貌应该夸一下，但不宜妄自菲薄。大家都是人类，没什么做得到做不到的。

​ 将来有时间了，还是想自己做个动态博客，多设计点儿有趣的功能。我身边有CSDN多次热榜第一的室友，有ACM拿过学校第一枚区域赛奖牌的同学，有大一就和中科大还是院来着一起搞论文的学弟。但对现阶段的我来说，再大的成就都比不上通过考研初试和复试。

常用命令

​ utteranc.es客户端地址：https://utteranc.es/client.js

​ 获取QQ头像的url: https://q1.qlogo.cn/g?b=qq&nk=QQ号码&s=640。用edge浏览器的时候发现，用https访问博客，如果头像url用http的话会被拦截。

​ 每次输入这么多命令太麻烦了，索性写了一个windows脚本上传博客。提前在path环境变量中配置hugo命令所在目录，以在任意位置执行hugo命令。

​ 还是第一次主动写windows脚本，上一次写还是上一次（呸，上次是改U盘图标，照网上抄的）。脚本文件后缀名为.bat，直接在里面写命令即可。

hugo -D 
cd ./public
git add -A
git commit -m "脚本提交"
git push -u origin master

写博客的技巧总结

  • 放点儿好康的图，开头和中间黑文字多的地方
  • 彩蛋永不过时
  • 对diary这个主题来说，描述不要写太长，不好看；
  • 做主题的还是放相关的图片比较好，你看那些杂七杂八、二次元美图成灾的博客，一点儿都不好看
